Appendix

Data Sources & Methodology

All data is read from a local SQLite database populated by syncing with the Toggl Track API v9 and Toggl Reports API v3. Entries with duration ≤ 0 (running timers) are excluded from all analyses.

  • CSV sync: fast historical backfill, ~1 API call per calendar year.
  • JSON enrichment sync: full field set including client, task, tag metadata.
  • Tags are stored as JSON arrays; native Toggl IDs used post-enrichment; SHA-256 synthetic IDs for CSV-only entries.

Analyzer Descriptions

  • Longitudinal: Stacked category composition, HHI concentration index, rolling stats, YoY heatmap, session violin, active-days rate, transition velocity.
  • Changepoints: PELT + Binseg change-point detection via ruptures across 5 signals; transition event aggregation; annotated timeline.
  • Rhythms: Hour-of-day heatmap by year, day-of-week patterns, sleep/wake proxy, seasonal decomposition, weekend ratio, consistency CoV.
  • Correlations: Pearson correlation heatmap, crowding-out analysis, trade-off pairs, K-Means week archetypes, lead/lag cross-correlation, network graph.
  • Text Mining: TF-IDF top terms, NMF + LDA topic modeling, topic prevalence over time, VADER sentiment drift, vocabulary evolution heatmap, description length trend.
  • Life Phases: Multivariate PELT on weekly feature matrix; auto-labeled phase epochs; Gantt chart, radar chart, phase comparison table.

Dependencies

  • Python 3.10+ · Pandas · NumPy · Plotly · Jinja2
  • scikit-learn (TF-IDF, NMF, LDA, KMeans, StandardScaler)
  • ruptures (PELT and Binseg change-point detection)
  • scipy (seasonal decomposition, statistical tests)
  • vaderSentiment (sentiment analysis)

Data Quality Notes

  • Enrichment data (client, task, full tag metadata) is only available after running a JSON enrichment sync and is lost on Streamlit Cloud cold starts.
  • Text mining results depend on description coverage — many entries may have no description.
  • Change-point sensitivity is set to pen=3 for PELT; adjust in analysis/analyzers/changepoints.py.
  • Life phase detection requires at least 52 weeks of data (1 year) for meaningful segmentation.
